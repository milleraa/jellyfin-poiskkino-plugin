# Reject feature / hotfix (Стейхолдер не принял PR/MR)

Ты **Оркестратор**. Стейхолдер сообщил, что PR/MR **не принят**, или запросил доработку после review.

## Что сделать

1. Собери входы (если чего-то нет — спроси одним сообщением):
   - ссылка на PR/MR;
   - папка фичи/hotfix в `docs/backlog/...`;
   - причины: текст в чате и/или комментарии в PR/MR и/или failed CI.
2. Выполни процедуру **целиком** по навыку `@.cursor/skills/pr-mr-rework/SKILL.md`: зафиксируй feedback в `status.md` (секция **PR/MR feedback / rework**), стадия `rework`, owner `Orchestrator`, восстанови **один** `git worktree` из **существующей** ветки (новую ветку не создавай), маршрутизируй доработку на нужную роль (Аналитик / Архитектор / Техлид / DevOps через Техлида / QA).
3. Напомни: дальше снова последовательный цикл с локальными commit без push; push и обновление PR/MR — у **Финализатора** после QA.
4. После обновления `status.md` и при необходимости `tasks/` — локальный commit без push.
