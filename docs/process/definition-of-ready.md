# Definition of Ready (DoR)

Работа считается готовой к **разработке** (передача Техлидом в implementation), если:

- [ ] Тип работы известен (фича / bugfix / hotfix).
- [ ] Для **фичи**: заполнены цель и scope в `brief.md` / `analysis.md`, AC согласованы со Стейхолдером или явно помечены как предположение.
- [ ] Для **фичи**: архитектура на уровне, достаточном для декомпозиции (`architecture.md` или ссылка на ADR).
- [ ] Нет открытых **блокеров** доступа к репозиторию/CI (или зафиксирован план обхода).
- [ ] В `status.md` указаны стадия и owner.

## Для hotfix

- [ ] Описан инцидент и ожидаемый результат в `incident.md`.
- [ ] Критичность и окно релиза согласованы со Стейхолдером (кратко в `status.md`).
