# Архитектурные принципы

- **Clean Architecture** — домен и use cases не зависят от инфраструктуры.
- **Инверсия зависимостей** — абстракции внутрь, адаптеры снаружи.
- **SOLID** — особенно SRP и DIP для границ модулей.
- **Явные границы** — API между контекстами документируются.
- **ADR** для значимых решений.
- **Явное** лучше неявного.

Правила агента: `.cursor/rules/50-architecture.mdc`.
